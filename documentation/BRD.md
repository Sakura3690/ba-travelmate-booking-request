# Business Requirements Document (BRD)
**Project Name:** Travelmate Booking Request

## 1. Мета документа (Document Purpose)
Цель данного документа — описать основные бизнес-требования к модулю «Заявка на бронирование» (Booking Request) для платформы Travelmate. Документ служит основой для понимания того, какой функционал должен быть реализован, и является ориентиром для команды разработчиков, тестировщиков и стейкхолдеров.

## 2. Рамки проекта (Scope)
**Входит в рамки проекта (In Scope):**
* Форма создания заявки на бронирование (выбор направления, дат поездки, количества людей, предпочтений по бюджету).
* Возможность для пользователя просматривать историю своих заявок и их текущий статус (New, Processing, Approved, Rejected, Canceled).
* Уведомления на email об изменении статуса заявки.
* Панель менеджера для просмотра входящих заявок, связи с клиентом и изменения статуса бронирования.

**Не входит в рамки проекта (Out of Scope):**
* Процесс онлайн-оплаты и интеграция с платежными шлюзами (будет реализовано в следующих фазах).
* Мобильное приложение (пока реализуется только веб-версия).
* Система отзывов и рейтингов.

## 3. Вимоги стейкхолдерів (Stakeholder Requirements)
* **Клиент (Traveler):** Должен иметь возможность интуитивно понятно и быстро создать заявку, а также отслеживать процесс ее обработки без необходимости звонить в поддержку.
* **Менеджер по бронированиям (Booking Agent):** Должен иметь единый интерфейс (дашборд) со списком всех новых заявок для быстрой их обработки, подтверждения или отклонения.
* **Владелец продукта (Product Owner):** Ожидает, что внедрение модуля заявок сократит время обработки запросов клиентов на 30% и снизит количество ошибок при ручном вводе данных.

## 4. Бізнес-правила (Business Rules)
1. Заявка не может быть создана на прошедшие даты.
2. При создании заявки система должна автоматически присваивать ей статус `New`.
3. Только авторизованные пользователи (с подтвержденным email) могут отправлять заявку на бронирование.
